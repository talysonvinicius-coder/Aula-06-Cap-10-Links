Explicação do View e do Controller

View (Visualização): É a camada de interface do usuário, responsável por exibir os dados para o usuário. A View recebe os dados processados pelo Model e os apresenta de forma visual, como telas ou tabelas, e também pode enviar ações do usuário para o Controller.

Controller (Controlador): É o mediador entre o Model e a View. Ele recebe as requisições e entradas do usuário (da View), aciona as ações necessárias no Model (lógica de negócio) e, por fim, seleciona a View apropriada para apresentar a resposta ao usuário.
